Google Consent Mode v2 for Magento 2

Published on: August 14, 2024

Looking to enhance user privacy on your Magento 2 website? Learn about Consent Mode v2 and how it can benefit your online business.

What is Google Consent Mode v2?

Google Consent Mode v2 is a solution that helps website owners manage user consent for tracking technologies in a privacy-centric way. This updated version offers flexibility in obtaining user permission for data usage.

It adjusts Google tags based on user interactions with permission prompts, allowing for the collection of approval for analytics while respecting user choices.

Businesses can optimize marketing and analyze performance without compromising privacy. A key feature is the ability to passively measure interactions even when users opt out of tracking, providing insights into traffic and engagement trends.

Magento 2 site owners can balance compliance with regulations like GDPR and CCPA and effective data-driven marketing. In summary, it empowers Magento 2 merchants to enhance user experiences while ensuring compliance.

How does Google Consent Mode v2 Work?

Magento 2 Google Consent functions by enabling website owners to adjust the behavior of their Google tags based on the consent status of users. This process is divided into two primary components: consent types and integration.

Consent Mode v2 supports different types of consent, including Ad Storage, Analytics Storage, etc.

Advantages of Using Google Consent Mode v2 Magento

Implementing Magento 2 Google Consent in your Magento 2 store offers several advantages that can significantly improve your business operations and customer satisfaction. Below are some key benefits:

  • Compliance with Privacy Laws: Ensures adherence to cookie regulations, particularly for customers in the European Economic Area (EEA), demonstrating respect for user data privacy.
  • Improved Advertising Efficiency: Bridges data gaps left by non-consenting users, allowing for more informed advertising decisions and optimizing ad spending.
  • User Trust Maintenance: Balances data collection with user privacy, fostering trust between businesses and their customers.
  • Accurate Campaign Performance Analysis: Helps businesses obtain clearer insights into campaign effectiveness, even with limited user consent.
  • Efficient Data Collection: Streamlines the data collection process while maintaining user privacy and complying with regulations.

How to Use Magento 2 GTM Consent Mode v2 Extension?

Follow these steps to effectively use the Magento 2 GTM Consent Mode V2 extension:

Step 1: Install the Extension

1. Download the Extension: Obtain the extension zip file from the official Meetanshi.com source.

2. Extract the Files: Unzip the downloaded file and place the extracted contents into your Magento root directory.

3. Run Installation Commands: Access your server via SSH and execute the following commands one by one:

  • php bin/magento setup:upgrade
  • php bin/magento setup:static-content:deploy -f
  • php bin/magento cache:flush

After completing these commands, the extension will be successfully installed on your Magento 2 platform.

Step 2: Customize the Cookie Banner

Access the Magento admin panel. Navigate to Stores > Configuration > Meetanshi > Cookie Consent V2 and enable the extension.

After activation, you can customize the cookie banner according to your preferences.

General Cookie Settings Users can modify their cookie preferences at any time using the cookie preferences button. You have the option to:

  • Display or hide this button
  • Alter its background color

Cookie Notice Customization The cookie notice appears to users on their first visit to the site. You can:

  • Modify the notice text
  • Customize the link
  • Set an expiration date for preference data

Customize the following buttons’ labels and colors:

  • Accept: Grants consent for all cookie types
  • Decline: Declines all non-essential cookies
  • Set Custom Preferences: Opens a pop-up where users can select specific cookie preferences

Step 3: Edit Cookie Consent Types

The extension includes pre-configured consent types for Google services like ads and analytics. These are organized into cookie groups that customers can accept or reject.

You can modify existing consent types and cookie groups or create new ones.

To manage cookie groups:

  1. Navigate to Cookie Consent V2 > Manage Cookie Groups
  2. Create or edit groups as needed

For each cookie group, you can:

  • Enable/disable the group (only enabled groups appear on the front end)
  • Mark as essential (making it mandatory)
  • Assign a unique code for reference
  • Set a display name for the preferences pop-up
  • Add a description
  • Adjust the sort order to control its position in the pop-up

Configuration options:

  • Set as Essential: When enabled, users cannot deny this group
  • Cookie Group Code: A reference code (typically the group name in lowercase)
  • Cookie Group Name: The name shown in the preferences pop-up
  • Description: Explains the purpose of the cookie group

After configuring, click Save to store your changes.

To manage consent types:

  1. Navigate to Cookie Consent V2 > Manage Consent Parameters
  2. Edit existing parameters or create new ones
  3. To add a new consent parameter, click the Add a New Consent Parameter button

You can then associate these consent types with the appropriate cookie groups.

Configure the following settings for each consent parameter:

  • Parameter Code: Enter the consent code to be sent to Google tag
  • Default Status: Choose the initial status for this consent type
  • Cookie Group: Select which cookie group this consent belongs to

Once you’ve finished configuring the consent parameter, click the Save Parameter button to apply your changes.

Demonstrating Magento 2 Consent Mode V2 Functionality

After configuring the extension, your Magento 2 store will have a fully functional cookie consent mode V2.

First-time visitors to your site will see the cookie consent notification banner.

The cookie status updates according to the user’s choices. If they select “custom preferences,” they can choose specific cookie groups from the available options. These options reflect the active cookie groups configured in the backend.

Once users set their preferences, the cookie consent banner won’t reappear until the expiration period (set in Step 1) has passed.

If enabled, a cookie preferences button will be visible, allowing users to modify their choices at any time.

To view a record of customer consent preferences, access the Cookie Consent V2 > Cookie Consent Logs grid in the admin panel.

Step 4: Test Google Consent Mode v2

To test Google Consent Mode v2 Magento 2, first ensure that the Magento 2 Google Consent Mode v2 extension is enabled and properly configured in your Magento settings.

Next, simulate user consent by interacting with the consent toggle on your website and monitor the Google Tag Manager for corresponding data signals.

You can use tools like Google Tag Assistant to verify if the consent settings are correctly applied and check if the appropriate events are firing based on the consent given or denied. Finally, review the data flow in Google Analytics to confirm that tracking behaves as expected with and without user consent.

How to Add Google Consent Mode v2 for Magento 2 using GitHub Code?

To implement Google Consent Mode v2 in Magento 2, you can use the Cookie Consent repository by jamestomasino on GitHub. Here’s a streamlined guide:

1. File placement:

  • Add cookieconsent.js to app/design/frontend/[Vendor]/[Theme]/web/js
  • Add cookieconsent.css to app/design/frontend/[Vendor]/[Theme]/web/css

2. Include files in default_head_blocks.xml:

<head>
<css src=”css/cookieconcent.css”/>
<script src=”js/cookieconcent.js”/>
</head>

3. Add consent trigger in header.phtml :

<button class="cookie-concent-banner-open">Change Cookie Consent </button>

4. Deploy theme changes and clear caches.

5. Test the consent banner and verify data transmission to GTM.

While this method is straightforward, it requires technical knowledge and may be prone to errors. Consider using a more robust solution for complex implementations.
