How Many types of Dependency Injection in Magento 2

Published on: February 01, 2023

How Many Types of Dependency Injection in Magento 2 and How it is Used in Magento 2: Automatic Dependency Injection (ADI) eliminates the need for an object to search for its dependent objects. There are three types of ADI: Constructor Injection, Setter Injection, and Interface Injection. Magento 2 uses ADI, specifically Constructor Injection, where dependencies are passed as parameters in the constructor to adhere to injection rules. This method is an alternative to the Mage Class method used in Magento 1. An example in Magento 2 is provided to illustrate this concept.

A Complete Guide to Types of Dependency Injection in Magento 2

The Helper Class in Magento 2, Data.php, is defined as follows:

namespace Namespace\Module\Helper;

class Data extends \Magento\Framework\App\Helper\AbstractHelper

{

    public function HelperFunc()

    {

        return 1;

    }

}

The HelperFunc() method can be called using Dependency Injection (DI) in the following way:

class ClassName

{

    public function __construct(\Namespace\Module\Helper\Data $helper)

    {

        $this->helper = $helper;

    }

    public function func()

    {

        $this->helper->HelperFunc();

    }

}

In the above code, the Dependency is injected into the constructor of the ClassName class. This method, which involves dependency injection in Magento 2, can be used throughout Magento 2 and is a powerful way to manage dependencies and decouple components.

Why Magento 2 Dependency Injection is Preferred Over Object Manager?

While the Object Manager in Magento 2 can instantiate objects and inject dependencies through its create() method and instance configuration files, it is not recommended to use it. If you’re wondering how many types of dependency injection in Magento 2 are available, it’s worth noting that Magento 2 supports three types of dependency injection: constructor injection, setter injection, and property injection. Using the Object Manager defeats the purpose of Dependency Injection, as it does not follow Magento 2’s development processes and can create hidden dependencies. It may be used in some parts of the core, but it is expected to be refactored in the future. Adhering to Magento 2’s preferred method of dependency management leads to a cleaner and more organized codebase, better performance, and improved scalability.
