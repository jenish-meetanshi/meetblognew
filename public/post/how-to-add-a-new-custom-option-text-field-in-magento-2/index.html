How to Add a New Custom Option Text Field in Magento 2.

Published on: February 14, 2023

If you’re running an e-commerce store using Magento 2, you may want to consider adding a custom option text field to your registration process. Doing so can be an effective way to gather additional information from your customers, whether you’re looking to collect a preferred nickname or data for targeted marketing campaigns. The custom option text field is both flexible and customizable, making it a versatile solution for your business needs.

One of the standout features of Magento 2 is the ability to create configurable product in Magento 2. By providing customers with a range of choices for specific products, you’re improving their shopping experience and potentially generating more revenue. If you haven’t enabled this feature yet, it’s worth considering to help you make your store more customized.

Thankfully, how to add a new custom option text field in Magento 2 is a fairly simple process. We’ve put together a step-by-step guide to walk you through it and help you get started.

What is Custom option text field 

In Magento 2, a custom option text field is a type of form field that enables customers to input text-based information when registering. This valuable feature can collect additional customer data such as preferred nicknames, marketing preferences, or any other relevant information.

To create and manage custom option text fields, you can access the Magento 2 Admin Panel where you can easily set the label, input type, and other properties for the field. Once created, the custom option text field will be displayed on the customer registration page, allowing customers to provide their information during the registration process. By gathering this data, you can enhance the customer experience and drive sales through targeted marketing campaigns and personalized promotions.

Important Factors to Consider 

In addition to the steps outlined above, here are some additional tips to consider when adding a new custom option text field in Magento 2:

Make the Field Required or Optional: 

You can set the “Values Required” option to “Yes” or “No” to determine whether customers are required to enter information in the custom option text field during registration.

Specify a Default Value: 

You can specify a default value for the custom option text field in the “Default Value” section. This value will be automatically populated in the field if a customer does not enter any information.

Provide a Description: 

To provide additional context for customers, you can add a description for the custom option text field in the “Storefront Properties” section.

Validate the Input: 

You can validate the input in the custom option text field by specifying validation rules. For example, you can set the input to be a number or a certain length.

Use the Field for Customer Segments: 

By setting the “Use in Customer Segment” option to “Yes”, you can use the information collected in the custom option text field to create customer segments and target specific groups of customers with personalized promotions and marketing campaigns.

By following these tips, you can ensure that your custom option text field is set up correctly and meets the needs of your store.

Steps to add a new custom option text field in Magento 2

SR/StackExchange/etc/adminhtml/di.xml

<?xml version="1.0"?>

<config xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:noNamespaceSchemaLocation="urn:magento:framework:ObjectManager/etc/config.xsd">

    <type name="Magento\Catalog\Ui\DataProvider\Product\Form\Modifier\CustomOptions">

        <plugin name="sr_stackexchange_custom_option" type="SR\StackExchange\Plugin\Catalog\Ui\DataProvider\Product\Form\Modifier\CustomOptions" sortOrder="1"/>

    </type>

</config>

SR/StackExchange/Plugin/Catalog/Ui/DataProvider/Product/Form/Modifier/CustomOptions.php

namespace SR\StackExchange\Plugin\Catalog\Ui\DataProvider\Product\Form\Modifier;

class CustomOptions

{

    public function afterModifyMeta(

        \Magento\Catalog\Ui\DataProvider\Product\Form\Modifier\CustomOptions $subject,

        $meta

    ) {

        $meta['custom_options']['children']['options']['children']['record']['children']['container_option']['children']['container_common']['children']['custom_text'] =

        $this->getTitleFieldConfig(

            200,

            [

                'arguments' => [

                    'data' => [

                        'config' => [

                            'label' => __('Custom Text'),

                            'component' => 'Magento_Catalog/component/static-type-input',

                            'valueUpdate' => 'input',

                            'imports' => [

                                'optionId' => '${ $.provider }:${ $.parentScope }.option_id'

                            ]

                        ],

                    ],

                ],

            ]

        );

        return $meta;

    }

    /**

     * Get config for "Title" fields

     *

     * @param int $sortOrder

     * @param array $options

     * @return array

     */

    protected function getTitleFieldConfig($sortOrder, array $options = [])

    {

        return array_replace_recursive(

            [

                'arguments' => [

                    'data' => [

                        'config' => [

                            'label' => __('Custom Text'),

                            'componentType' => \Magento\Ui\Component\Form\Field::NAME,

                            'formElement' => \Magento\Ui\Component\Form\Element\Input::NAME,

                            'dataScope' => 'custom_text',

                            'dataType' => \Magento\Ui\Component\Form\Element\DataType\Text::NAME,

                            'sortOrder' => $sortOrder,

                            'validation' => [

                                'required-entry' => false

                            ],

                        ],

                    ],

                ],

            ],

            $options

        );

    }

}
