How to Add URL Rewrite Programmatically in Magento 2

Published on: February 15, 2023

Magento 2 is a popular e-commerce platform that offers an array of features and functionalities for merchants to manage their online store. One crucial aspect of an online store is the URL structure, which can impact the search engine rankings and user experience. URL rewriting is a feature of Magento 2 that enables merchants to create custom, SEO-friendly URLs for their products, categories, and other pages. Custom URLs can improve the search engine rankings, increase brand recognition, and provide a better user experience for customers.Â 

In this blog, we will delve into the process of how to add URL rewrite programmatically in Magento 2, exploring the steps and benefits of this process. You can also read, how to effectively use URL rewrite Functionality in Magento 2.

Why Add URL Rewrite Programmatically in Magento 2?

Adding URL rewrites programmatically in Magento 2 offers several benefits, including:

Improved SEO: 

Custom, SEO-Friendly URLs can improve your search engine rankings and increase visibility for your online store.

Better User Experience: 

Custom URLs make it easier for customers to find the products they’re looking for, leading to a better overall experience.

Increased Brand Awareness: 

Custom URLs that reflect your brand and product offerings can help to increase brand awareness and recognition.

How to Add URL Rewrite Programmatically in Magento 2:

To add URL rewrites programmatically in Magento 2, follow these steps:

1. Open your Magento 2 store’s code editor and navigate to the location where you want to add the URL rewrite.

2.  Create a new URL rewrite object using the following code:

bash

$rewrite = $this->urlRewriteFactory->create();

1. Set the parameters for the URL rewrite. For example, to create a URL rewrite for a product, you can use the following code:

scss

$rewrite->setStoreId(1)

        ->setIsSystem(0)

        ->setIdPath('product/123')

        ->setTargetPath('catalog/product/view/id/123')

        ->setRequestPath('product-123.html')

        ->setDescription('Product 123');

Note: In the above example, “product/123” is the ID path, “catalog/product/view/id/123” is the target path, “product-123.html” is the request path, and “Product 123” is the description.

    1. Save the URL rewrite using the following code:

scss

$rewrite->save();
