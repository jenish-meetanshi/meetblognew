How To Configure a Cron Job in Magento 2 admin: A Comprehensive Tutorial for Effective Performance.

Step 1: Navigate to Stores > Settings > Configuration

The first step in setting up a cron job in Magento 2 is to navigate to the store settings. From the main menu, go to Stores > Settings > Configuration.

Step 2: Open Advanced settings > System

Once you are in the Configuration section, you will see several options on the left sidebar. Click on Advanced settings and then select System.

Step 3: Open the Cron section

Under the System tab, you will see a section for Cron. Click on this section to access the cron job settings.

Step 4: Complete the settings in the Index and Default sections

In the Cron section, you will see two sections for settings: Index and Default. In both sections, you will need to complete the following options:

  • Generate Schedules Every: This determines the frequency of schedule creation in minutes.
  • Schedule Ahead for: This determines how early cron jobs will be scheduled.
  • Missed if not Run Within: If a cron job has not started after a specified time, it will not be started and will be marked as missed.
  • History Cleanup Every: This defines the time for cleaning up the history of ended tasks.
  • Success History Lifetime: Sets the history lifetime for the Successful status.
  • Failure History Lifetime: Sets the history lifetime for the Error status.
  • Use Separate Process: Choose whether to run a separate process for all cron jobs in the group.

Step 5: Save the config after completing

After completing the settings for scheduled tasks in the Magento 2 admin panel, it is crucial to saving the changes by clicking on the “Save Config” button.
