How to Configure Product Catalog Search in Magento 2?

Published on: March 21, 2023

Enhancing the product search experience in Magento 2 can greatly benefit your customers by allowing them to quickly find what they’re looking for through the use of the product search form.

Combined with layered navigation, Magento 2’s catalog search provides a seamless shopping experience and flexible product catalog navigation.

The search form is the most commonly utilized form of catalog navigation and can be accessed simply by typing in a word or phrase.

Additionally, customers who utilize the search form are more likely to make a purchase, making it a crucial aspect of the shopping experience.

In this article, we will explore How to Configure Product Catalog Search in Magento 2, delving into the various types of catalog search available and providing step-by-step instructions for its configuration. 

How to Configure Product Catalog Search in Magento 2: Streamlining the Shopping Experience

Magento 2 Catalog Search Types

For optimal customer experience on your Magento 2 store, it’s important to utilize the two available catalog search options – Quick Catalog Search and Catalog Advanced Search.

Quick Catalog Search:

With Magento 2’s Quick Catalog Search, customers can easily find the product they’re looking for by simply entering a few characters or a phrase that aligns with the product name or other product information.

Advanced Catalog Search:

To configure catalog search in Magento 2, you can take advantage of the Advanced Catalog Search feature, which allows customers to conduct more specific product searches. In addition to searching by product name, customers can also search by SKU, price, and long and short descriptions.

To further enhance the advanced search feature, you can add product attributes such as color and size to achieve more precise search results. For more information on how to customize this feature, refer to the “Storefront Properties” section of the product attribute configurations.

Magento 2’s advanced catalog search helps customers find products that best match their search criteria, improving their shopping experience.

By adjusting various catalog search configurations such as store search recommendations, suggestions, results count, and other relevant options, you can further improve the effectiveness of the search feature.

Setting Up Catalog Search in Magento 2

To configure catalog search in Magento 2:

1.Navigate to Stores > Configuration > Catalog > Catalog > Catalog Search.

2. Specify the minimum and maximum query lengths, which determine the range of characters allowed in a search query. These settings should align with the configurations of your chosen search engine, which will be discussed later in the article.

Note: It’s recommended to set a higher minimum query length for optimal performance.

3. Define the number of top search results to cache for quicker response times. The default is 100, but you can also set it to 0 to cache all search terms and results entered during subsequent searches.

4. Choose the search engine to use for catalog search.

Magento 2 offers two options for catalog search engines: MySQL and Elasticsearch.

All of the previously mentioned configurations will apply to both engines.

However, each engine has its own specific options that need to be configured once you make your selection.

MySQL Catalog Search in Magento 2

Configuring MySQL Catalog Search in Magento 2:

1. Set the Autocomplete Limit, which is the maximum number of search results displayed automatically.

2. Turn on Search Recommendations and specify the Search Recommendations Count to limit the number of recommendations shown when the customer enters a search query.

3. Choose to show or hide the Results Count for Each Recommendation, which displays the number of products matching the query in brackets next to each recommendation.

4. Enable Search Suggestions and set the Search Suggestions Count to limit the number of suggestions shown to the customer.

5. Opt to display or hide the Results Count for Each Suggestion, which shows the number of products in brackets next to each suggestion.

Note: Enabling both Search Recommendations and Suggestions can slow down website performance.

Setting Up Elasticsearch for Magento 2 Catalog Search

An alternative to the default MySQL search engine is Elasticsearch, a highly reputable solution utilized by many successful online stores. It is recommended to use Elasticsearch 6. x, as versions 2. x and 5. x are no longer supported.

Here’s how you can configure Elasticsearch for your Magento 2 catalog search:

1. Activate the EAV Indexer (only available for Elasticsearch or Elasticsearch 5. x).

2. Determine the maximum number of search results displayed automatically using the Autocomplete Limit.

3. Configure the Elasticsearch Connection:

  •  Specify the Elasticsearch Server Hostname, Port, and Index Prefix.
  •  Decide whether to enable Elasticsearch HTTP Auth to request a username and password for server access.
  •  Define the Elasticsearch Server Timeout.
  •  Test the Connection to ensure it’s properly established.

4. Similar to the MySQL search engine, set up the Search Recommendations and Suggestions for Elasticsearch.

Once you’ve completed the configurations, don’t forget to save them by clicking the Save Config button.
