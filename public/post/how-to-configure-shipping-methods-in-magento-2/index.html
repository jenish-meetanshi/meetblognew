How to Configure Shipping Methods in Magento 2?

Published on: March 30, 2023

If you are running an eCommerce store on Magento 2, you need to be familiar with its shipping methods. Shipping methods refer to the cost and mode of delivery for a product to the customer. Offering multiple shipping methods can enhance the customer experience and even influence their purchasing decisions. Hence, it is essential to understand the basic shipping configurations in Magento 2 and the default shipping methods available.

In this article, we will discuss how to configure shipping methods in Magento 2. We will cover the different types of shipping methods categorized based on delivery type, and provide an overview of the default shipping methods available in Magento 2. This guide will help you to understand and manage the Magento 2 shipping methods effectively to improve your eCommerce store’s shipping options.

How to Configure Shipping Methods in Magento 2: A Step-by-Step Guide to Optimizing Your Store’s Delivery Options

How to Configure Magento 2 Shipping Settings?

In order to effectively configure the default shipping methods available in Magento 2, it’s necessary to first set up the default shipping settings. This can be done by navigating to Stores > Configuration > Sales > Shipping Settings.

Here, you’ll need to configure the Origin and Shipping Policy Parameters, which will allow Magento to calculate taxes and shipping costs based on the shipping origin.

The information you’ll need to provide includes:

  • Country
  • Region/State
  • ZIP/Postal Code
  • City
  • Street Address/Street Address Line 2

Additionally, you can choose to Apply a custom Shipping Policy and provide the text for it, which will be displayed during checkout.

How to Configure Default Magento 2 Shipping Methods?

Once you finish with the basic shipping settings, you’re good to go and configure shipping methods. They are managed in Stores > Configuration > Sales > Shipping Methods.

Flat Rate Shipping Method

Flat Rate is the standard shipping method in Magento 2, which can be applied either per single item or per order, regardless of the product attributes such as size or weight.

There are some common options for every shipping method, such as Title, Enable/Disable, and Method Name, but let’s focus on the unique ones.

1. In the Type field, you can define how the shipping fee is added to the subtotal. There are three options: None, Per Item, and Per Order.

  •  None means no charge is applied, making a Flat Rate equivalent to Free transporting.
  •  Per Order means the charge is applied to the entire order, not each item separately.
  •  Per Item means the Flat Rate shipping fee is set for each item and multiplied by the number of items in the cart.

2. Set the Price you would like to charge for the Flat Rate Shipping.

3. Determine the method for Calculating the Handling Fee.

4. Set the amount of the Handling Fee based on the method you selected for handling fee calculation. For example, if you choose to calculate the handling fee from the order subtotal as a percentage, you would enter 0.08 for 8%.

5. Fill in the Displayed Error Message field with the text customers will see when the Flat Rate Shipping is unavailable.

6. In the Ship to Applicable Countries select box, choose whether you want to ship to all allowed countries or select specific countries.

7. Decide if you want to Show Method if Not Applicable, which determines if the Flat Rate shipping method will be displayed during checkout even if it can’t be applied to the order.

8. Set the Sort Order to determine the place of the Flat Rate method in the list of shipping methods.

Free Shipping Method

Free shipping is one of the most popular and customer-attracting shipping types used in Magento 2. It provides merchants with the ability to offer customers free shipping based on a minimum order amount and to set up cart price rules if the conditions are met.

The configuration for free transporting is similar to the previous method, however, you can delve deeper into the free shipping configuration to learn how to effectively promote it.

Table Rates Shipping Method

The Table Rates shipping method in Magento 2 calculates shipping fees based on various conditions, such as weight versus destination, price versus destination, and the number of items versus the destination.

This means that the cost of Table Rates shipping can vary based on the customer’s address and the number of items they order, or the weight of the items.

While Table Rates are similar to the Flat Rate shipping method, it has some distinct features, such as:

1. Select the condition rates will be calculated under.

2. Determine if virtual products should be included in price calculation with the “Include Virtual Products in Price Calculation” option. Note: virtual products do not have a weight, so they will not affect the results of the weight versus destination condition calculation.

To complete the default Table Rates configuration, you will need to prepare or update the table rate files.

3. In the Stores > Configuration section, switch to the desired store view or website to apply this shipping setting. Two new options will appear.

4. Export the CSV file, make any necessary updates, and then import it.

5. Visit the storefront and make several orders with different destinations to see how the Table Rates shipping fees are calculated.

Dimensional Weight Shipment in Magento 2

The Magento 2 Dimensional Shipping method calculates shipping rates based on the dimensions (volume, weight) or amount of space occupied by the package.

Here are the steps to configure the Dimensional Shipping method:

1. Enable DHL for checkout to display it during the checkout process.

2. Give the shipping method a Title and enter the Access ID, Password, and Account Number.

3. Set the Content-Type (Non-Domestic).

4. Manage the Handling Fee by specifying the type of calculation, the amount applied, and the handling fee amount.

5. Choose to divide the order weight into smaller units to accurately calculate the shipping charge.

6. Set the weight measurement units and size for the calculations.

7. Select the supported Documents Allowed Methods and Non Documents Allowed Methods.

8. Specify the ready time in the Ready Time field, indicating when the package will be ready for pickup after the order is placed.

9. Fill in the Displayed Error Message field with the text customers will see when DHL is unavailable.

10. Choose the preferred shipping method for free shipping offers using the Free Method setting.

11. Enable the Free Shipping Threshold to allow for free DHL shipping for orders that meet the minimum amount.

12. Configure the applicable countries for shipping, visibility, and sort order in the same manner as the other shipping methods.

13. Enable Debug to create a log file with error information.

14. Enable the Sandbox Mode to run DHL transactions in a test environment.

After you configure the shipping methods, don’t forget to Save Config.
