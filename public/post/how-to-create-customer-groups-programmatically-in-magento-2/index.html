How to Create Customer Groups Programmatically in Magento 2 

Published on: April 04, 2023

Magento 2 is a highly customizable e-commerce platform that enables you to create an online store that aligns with your unique business requirements. One of the many powerful features of Magento 2 is the ability of Configuring Magento 2 customer accounts with customer groups. Customer groups allow you to segment your customers based on various criteria, such as purchase history, location, or website behavior, enabling you to offer personalized experiences to each group, including tailored product recommendations, discounts, or promotions.

Programmatically creating customer groups in Magento 2 is a crucial feature for developers and store owners who want to streamline the process of creating customer groups. This can save time and reduce the likelihood of errors that can occur when creating customer groups manually through the admin panel. By creating customer groups programmatically, you can ensure that all groups are created consistently, with the same settings and attributes, which can help you maintain a more organized and efficient customer management system.

In this tutorial, we’ll take you through the steps of creating customer groups programmatically in Magento 2, using a custom module and the Group Factory class. Whether you’re a developer aiming to integrate this feature into your online store or a store owner seeking to enhance your customer management system, this tutorial has got you covered. You’ll find all the necessary steps to get started with creating customer groups programmatically in Magento 2.

Steps to Create Customer Groups Programmatically in Magento 2

Step 1: Create a Module

The first step is to create a module to contain our custom code. To do this, create a new directory in the app/code directory with the name of your module. For example, if you want to name your module “Custom”, you would create the directory app/code/Custom.

Inside this directory, create a registration.php file with the following contents:

php

<?php

\Magento\Framework\Component\ComponentRegistrar::register(

    \Magento\Framework\Component\ComponentRegistrar::MODULE,

    'Custom_CustomerGroup',

    __DIR__

);

This registers your module with Magento.

Next, create a module.xml file with the following contents:

xml

<?xml version="1.0"?>

<config xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:noNamespaceSchemaLocation="urn:magento:framework:Module/etc/module.xsd">

    <module name="Custom_CustomerGroup" setup_version="1.0.0" />

</config>

This defines the name and version of your module.

Step 2: Create a Customer Group

Now that we have our module set up, we can create a customer group programmatically. To do this, create a new file in the Custom/CustomerGroup directory called InstallData.php with the following contents:

php

<?php

namespace Custom\CustomerGroup\Setup;

use Magento\Customer\Model\GroupFactory;

use Magento\Framework\Setup\InstallDataInterface;

use Magento\Framework\Setup\ModuleContextInterface;

use Magento\Framework\Setup\ModuleDataSetupInterface;

class InstallData implements InstallDataInterface

{

    private $groupFactory;

    public function __construct(GroupFactory $groupFactory)

    {

        $this->groupFactory = $groupFactory;

    }

    public function install(ModuleDataSetupInterface $setup, ModuleContextInterface $context)

    {

        $setup->startSetup();

        // Create customer group

        $groupName = 'My Group';

        $groupCode = 'my_group';

        $taxClassId = 3;

        $this->groupFactory->create()

            ->setCode($groupCode)

            ->setTaxClassId($taxClassId)

            ->setCustomerGroupCode($groupName)

            ->save();

        $setup->endSetup();

    }

}

This file defines a class that implements the Install Data Interface interface. This interface provides a method called install(), which is called during the installation of the module. In the install() method, we create a new customer group using the Group Factory class. We set the name, code, and tax class ID for the group, and then save it using the save() method.

In this example, we have set the name of the group to “My Group”, the code to “my_group”, and the tax class ID to 3. You can change these values to suit your needs.

Step 3: Run the Setup Script

To run the setup script and create the customer group, we need to run the following command in the root directory of your Magento 2 installation:

bash

php bin/magento setup:upgrade

This will run the setup script for all installed modules, including our custom module. Once the script has completed, the new customer group should be available in the Magento 2 admin panel.
