How To Disable Magento 2 Profiler

Published on: April 05, 2023

Magento 2 is a popular e-commerce platform that empowers online businesses to create and manage their e-commerce websites. One of its notable features is the profiler, which is a tool that assists developers in optimizing the performance of their Magento 2 installation. Nevertheless, the profiler can potentially hinder a website’s performance, and therefore, disabling it may become essential. This article aims to guide you on how to disable Magento 2 profiler.

What is  Profiler in Magento 2

– The profiler in Magento 2 is a built-in feature that provides developers with valuable insights into their website’s performance. It enables developers to pinpoint areas where website speed and user experience may be affected, such as bottlenecks or slow-loading areas.

– The profiler generates detailed information on page load times, including the time taken to execute each process that makes up the page. Additionally, the profiler can also track the time it takes for media, such as images, to load as the user scrolls down the page by enabling lazy loading in Magento 2.

– However, the profiler can also have a negative impact on website performance by adding extra processing time and generating large log files that consume server resources. Disabling the profiler can help to speed up website performance and reduce the use of server resources, which is especially beneficial for high-traffic websites or those hosted on shared servers with limited resources.

– However, it’s important to note that disabling the profiler can also make it more challenging to troubleshoot and optimize the website in the future, as it removes a vital diagnostic tool for developers. Therefore, website owners and developers should carefully consider the tradeoffs of disabling the profiler before making any changes. It’s recommended to consult with a Magento 2 expert or perform some performance testing before making any decisions.

Steps to Disable Magento 2 Profiler

Step 1: Disable the profiler in the Magento 2 admin panel

The first step to disabling the Magento 2 profiler is to do so in the admin panel. You can do this by navigating to Stores > Configuration > Advanced > Developer > Debug and setting “Enabled Template Path Hints for Storefront” and “Enabled Template Path Hints for Admin” to “No”.

Step 2: Disable the profiler in the Magento 2 configuration file

The second step to disabling the Magento 2 profiler is to do so in the configuration file. You can do this by opening the app/etc/env.php file in a text editor and adding the following code to the file:

php

'debug' => 

  array (

    'profiler' => 

    array (

      'enabled' => false,

    ),

  ),

This code disables the profiler by setting the “enabled” value to false.

Step 3: Disable the profiler in the Magento 2 index.php file

The third step to disabling the Magento 2 profiler is to do so in the index.php file. You can do this by opening the index.php file in a text editor and adding the following code to the file:

php

if ($_SERVER['MAGE_PROFILER']) {

    \Magento\Framework\Profiler::disable();

}

This code disables the profiler by checking the MAGE_PROFILER environment variable and disabling the profiler if it is set.

Step 4: Clear the cache

After making these changes, it is important to clear the Magento 2 cache to ensure that the changes take effect. You can do this by running the following command from your Magento 2 root directory:

python

php bin/magento cache:clean

Conclusion

Disabling the Magento 2 profiler can greatly enhance the performance of your e-commerce website. By following the steps provided, disabling the profiler can be done quickly and easily. It is essential to keep in mind, however, that while this can lead to better performance, it may also impede future debugging and optimization. Therefore, careful consideration should be given to the effects of disabling the profiler before taking action.
