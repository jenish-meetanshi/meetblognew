The following steps will guide you on how to install Magento 2.4 in Ubuntu 20.04.

Prerequisites:-

  • Ubuntu 20.04
  • Root SSH access. Or regular user with sudo privileges

How to Install Magento 2.4 in Ubuntu: A Comprehensive Guide

Steps to Install Magento 2.4 on Ubuntu 20.04

1. Update your Ubuntu System :

sudo apt update

sudo apt upgrade

This will ensure that your system has the latest security updates and bug fixes.

2. Install the LAMP stack (Apache, PHP, and MySQL) by running the following command:

sudo apt install apache2 mysql-server php php-common php-mysql php-xml php-intl php-mbstring

This will install the Apache web server, the MySQL database server, and the PHP programming language.

3. Enable mod_rewrite for Apache:

sudo a2enmod rewrite

mod_rewrite is an Apache module that allows you to manipulate URLs and perform redirects. It is required for Magento to function properly.

4. Restart Apache:

sudo systemctl restart apache2

This will ensure that the changes you made to Apache take effect.

5. Create a new MySQL user and database for Magento:

mysql -u root -p

    CREATE USER 'magento'@'localhost' IDENTIFIED BY 'password';

    CREATE DATABASE magento;

    GRANT ALL PRIVILEGES ON magento.* TO 'magento'@'localhost';

     FLUSH PRIVILEGES;

    exit;

 This will create a new MySQL user named ‘Magento’ with the password ‘password’, and a new database named ‘Magento’. The user will have all privileges on the database.

6. Download the Magento 2.4.4 archive from the official website and extract it to the Apache document root:

sudo wget https://github.com/magento/magento2/archive/2.4.4.tar.gz -P /var/www/html/

sudo tar -zxvf /var/www/html/2.4.4.tar.gz -C /var/www/html/

This will download the Magento 2.4.4 archive to the /var/www/html/ directory and extract it.

7. Set correct permissions for the Magento files:

sudo chown -R www-data:www-data /var/www/html/magento2-2.4.4

This will give the Apache web server user (www-data) ownership of the Magento files.

8. Create a new virtual host file for Magento:

sudo nano /etc/apache2/sites-available/magento.conf

<VirtualHost *:80>

    ServerName magento.local

    DocumentRoot /var/www/html/magento2-2.4.4

    <Directory /var/www/html/magento2-2.4.4>

        Options Indexes FollowSymLinks

        AllowOverride All

        Require all granted

    </Directory>

</VirtualHost>

This will create a new virtual host file for Magento, which tells Apache to serve content from the /var/www/html/magento2-2.4.4 directory when a client requests the “Magento.local” hostname.

9. Enable the new virtual host:

sudo a2ensite magento.conf

This will enable the new virtual host file you just created.

10. Restart Apache:

sudo systemctl restart apache2

This will ensure that the changes you made to Apache take effect and the new virtual host is loaded.

11. Open your web browser and navigate to http://magento.local to start the Magento installation process. You will be prompted to agree to the terms and conditions and then asked to provide your database details, admin account information, and store settings.

12. Follow the on-screen instructions to complete the installation. This includes providing your database details, admin account information, and store settings.

In the Database Connection section, enter the following details:

– Database Server Host: localhost

– Database Server Username: Magento

– Database Server Password: password

– Database Name: Magento

In the Web Configuration section, enter the following details:

– Base URL: http://magento.local/

– Admin URL: http://magento.local/admin

In the Customize Your Store section, enter the following details:

– Store Name: My Magento Store

– Store Email: admin@example.com

– Admin User: admin

– Admin Password: password

Once you have provided all the required information, click the Install Now button to begin the setup Magento 2 process. After the installation is completed successfully, you will be redirected to the Magento admin panel.

Note:

  • You should also need to add ‘magento.local’ in your host file, it should be ‘/etc/hosts’ in Ubuntu.
  • You should also need to configure the php.ini file, to increase the memory limit, upload size, and other settings as per your requirement.
  • It is always recommended to install an SSL certificate and configure it to your Magento website.
  • Always take a backup of your website and database before making any major changes.

Conclusion

To install Magento 2.4.4 on Ubuntu 20.04, you need to follow several steps carefully. First, update your system and install the LAMP stack. You also need to enable mod_rewrite and create a MySQL user and database.

After that, download and extract the Magento archive, set file permissions, create a virtual host, and configure Magento installation via a web-based setup wizard. However, it’s crucial to update the host file, configure the php.ini file, install an SSL certificate, and back up your website and database before making any changes.

By following these steps, you can successfully install Magento 2.4.4 on Ubuntu 20.04 and learn how to install Magento 2.4 in Ubuntu. Keep in mind that paying close attention to each step is essential to ensure a successful installation process.

However, it’s worth mentioning that these steps are specific to Ubuntu 20.04, if you are looking to install Magento 2 on Centos 7, then different steps will be required.
