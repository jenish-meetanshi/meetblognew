You can install Magento sample data in three ways:

1. Using the Magento Command Line Interface (CLI): This method allows you to install sample data using the Magento CLI command.

2. Using Composer: This method uses the Composer package manager to install sample data.

3. By cloning the Magento GitHub repository: This method involves cloning the Magento sample data repository from GitHub and installing it manually.

The Magento CLI and Composer installation methods can be used when you’ve installed Magento 2 using an archive file from Adobe or a Composer meta package. However, if you’ve installed Magento by cloning its GitHub archive, these methods will not work. This is because the composer.json file in the GitHub archive doesn’t contain the “https://repo.magento.com” URL. In this case, you’ll need to install sample data by cloning its GitHub repository.

For complete removal of the Magento 2 sample data module, drop the database and reinstall Magento 2 with a new database.

Guide to Installing Sample Data in Magento

(1) Log in to your Server and Enable Developer Mode.

Access your server using SSH, switch to the Magento file system user, navigate to the Magento root directory, and enable developer mode by executing the command:

$ php bin/magento deploy:mode:set developer

After enabling developer mode, execute the command to clear any previously generated classes and proxies:

$ rm -rf generated/code/* generated/metadata/*

(2) Run the Sample Data Installation Command

To install the Magento 2 sample data module, execute the command:

$ php bin/magento sampledata:deploy

Note: You may be prompted to enter your Magento authentication keys at this point. If so, refer to Adobe’s guide on how to obtain your public and private keys, and use the same keys for the sample data module installation.

(3) Update the Database and Schema and Clear the Magento cache.

Execute the command to update the database and schema after the Magento 2 sample data module is successfully installed: 

$ php bin/magento setup:upgrade

Execute the following command to clear all Magento cache types:

$ php bin/magento cache:clean

$ php bin/magento cache:flush

(4) Verify The Sample Data Installation :

by checking if the sample data has been added to the Magento database, and by visiting the front of your Magento store to see if the sample data is being displayed correctly.

Finally, check your storefront by accessing it in a web browser to verify the sample data installation.

Note: It is important to remember that sample data should not be installed on a production server, as it cannot be uninstalled without deleting the Magento database and reinstalling Magento.

Troubleshooting Common Sample Data Installation Errors in Magento

Here are some common errors that may occur during the installation of Magento 2 sample data and the steps to resolve them:

1. File System Permissions Error: 

One of the most common errors encountered during the installation of Magento 2 sample data is a file system permission error. This can include exceptions such as “Can’t create directory /var/www/html/magento2/generated/code/Magento/CatalogRule/Model/.” or “Class Magento\CatalogRule\Model\RuleFactory does not exist.” To fix this, reset file system permissions based on your deployment and re-run the sample data: deploy command.

2. Magento Mode Error: 

Installing Magento sample data in production mode can cause a PHP Fatal error or the following error when you run the setup: upgrade command: “Production mode error Magento 2 sample data installation.” To prevent this, switch to developer mode and clear generated data before installing Magento 2 sample data.

3. PHP max_execution_time Error:

 The Magento sample data module installation process can be resource-intensive and may take several minutes. If the max_execution_time in your PHP configuration file is inadequate, the sample data script may fail silently. To fix this, switch to a user with root privileges, edit the php.ini file and increase the max_execution_time value to 600 or more. Remember to reset the max_execution_time value once the Magento 2 sample data installation is completed.
