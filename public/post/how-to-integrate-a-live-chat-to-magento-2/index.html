How to Integrate a Live Chat to Magento 2?

Published on: February 01, 2023

Every eStore owner knows the importance of Live Chat to their site, but perhaps using just words cannot describe how important it is to integrate Live Chat in M2. It provides Real-time Customer Support, boosts sales, and generates customer behavior data for further study. Nowadays, there are plenty of Live Chat services for you to choose from. In this article, we will guide you through the process of how to integrate a live chat to Magento 2 sites.

Simple Guide: How To Integrate A Live Chat To Magento 2

Step 1: Establish a Module

In this step, we establish a module named “LiveChat”.

To set up the module, you must create a registration file located at: your_magento_base/app/code/Webguru/LiveChat/registration.php

Note: Webguru is the company name used in this example. You can use any name that suits your needs.

<?php

\Magento\Framework\Component\ComponentRegistrar::register(

    \Magento\Framework\Component\ComponentRegistrar::MODULE,

    'Webguru_LiveChat',

    __DIR__

);

Then create: your_magento_base/app/code/Webguru/LiveChat/etc/module.xml

<?xml version="1.0"?>

<config xmlns:xsi="//www.w3.org/2001/XMLSchema-instance"

xsi:noNamespaceSchemaLocation="urn:magento:framework:Module/etc/module.xsd">

<module name="Webguru_LiveChat" setup_version="1.0.0">

</module>

</config>

Open your terminal, type:       

cd /var/www/html/magento2.1.6/

Please note that this guide uses M2 version 2.1.6, and the folder path may vary for different versions.

Then type: 

sudo bin/magento setup:upgrade

and  

sudo chmod -R 777 var pub

Go to your phpmyadmin, if there is “Webguru_LiveChat” in setup_module, your module is installed.

Step 2: Create a menu

Create: your_magento_base/app/code/Webguru/LiveChat/etc/adminhtml/menu.xml

<?xml version="1.0"?>

<config xmlns:xsi="//www.w3.org/2001/XMLSchema-instance" xsi:noNamespaceSchemaLocation="urn:magento:module:Magento_Backend:etc/menu.xsd">

    <menu>

        <add id="Webguru_LiveChat::menu"

             title="Live Chat"

             module="Webguru_LiveChat"

             sortOrder="25"

             resource="Magento_Backend::system" />

        <add id="Webguru_LiveChat::memberMenu"

             title="Live Chat Setting"

             module="Webguru_LiveChat"

             sortOrder="10"

             parent="Webguru_LiveChat::menu"

             resource="Webguru_LiveChat::config_livechat" />

        <add id="Webguru_LiveChat::configuration"

             title="Configuration"

             module="Webguru_LiveChat"

             sortOrder="15"

             action="adminhtml/system_config/edit/section/Webguru_livechat/"

             parent="Webguru_LiveChat::memberMenu"

             resource="Webguru_LiveChat::config_livechat" />

    </menu>

</config>

In the terminal, type:    

sudo rm -rf var

Step 3: Configure the Settings

In the previous step, you successfully created a menu. Now, it’s time to set up the configuration.

action=”adminhtml/system_config/edit/section/Webguru_livechat/”

In order to go to the configuration, we need to set up the configuration for that link.

Create: your_magento_base/app/code/Webguru/LiveChat/etc/adminhtml/system.xml

<?xml version="1.0"?>

<config xmlns:xsi="//www.w3.org/2001/XMLSchema-instance"

        xsi:noNamespaceSchemaLocation="urn:magento:module:Magento_Config:etc/system_file.xsd">

    <system>

        <tab id="Webguru" translate="label" sortOrder="200">

            <label>Webguru</label>

        </tab>

        <section id="Webguru_livechat" translate="label" type="text" sortOrder="80" showInDefault="1" showInWebsite="1" showInStore="1">

            <class>separator-top</class>

            <label>Live Chat</label>

            <tab>Webguru</tab>

            <resource>Webguru_LiveChat::config_livechat</resource>

            <group id="livechat" translate="label" type="select" sortOrder="30" showInDefault="1" showInWebsite="1"

                   showInStore="1">

                <label>Live Chat</label>

                <field id="enabled" translate="label comment" type="select" sortOrder="10" showInDefault="1"

                       showInWebsite="1" showInStore="1">

                    <label>Enable</label>

                    <source_model>Magento\Config\Model\Config\Source\Yesno</source_model>

                </field>

                <field id="widget" translate="label" type="text" sortOrder="20" showInDefault="1" showInWebsite="0"

                       showInStore="0">

                    <label>Widget</label>

                    <validate>required-entry</validate>

                    <depends>

                        <field id="enabled">1</field>

                    </depends>

                </field>

            </group>

        </section>

    </system>

</config>

In the terminal, type:  

sudo rm -rf var

From the live chat menu, click LIVE CHAT > Configuration

The configuration section has been successfully set up.

Step 4: Obtain the Widget Code from a Live Chat Service

In this tutorial, we use Chat Pirate as an example, but the process should be similar to other live conversation services.

1. Sign up for an account on chatpirate.com and access the dashboard.

2. Navigate to Settings > Integrations > Website

3. Copy the code and paste it into the Widget field in the Live Chat configuration.

4. Click “Save Config” and refresh the cache through Cache Management by selecting all and clicking “Submit refresh.”

If you have the widget code, integrating a chat to M2 is easy. Learn how to integrate a live chat to Magento 2 with our comprehensive guide. Then, complete the final step to enhance customer support and satisfaction.

Step 5: Implement on the Front-End

Retrieve the Widget Code from the Configuration established in the previous step.

Create: your_magento_base/app/code/Webguru/LiveChat/Block/Adminhtml/LiveChatHelper/LiveChatConfig.php

<?php

namespace Webguru\LiveChat\Block\Adminhtml\LiveChatHelper;

class LiveChatConfig extends \Magento\Framework\View\Element\Template

{

    protected $scopeConfig;

    const XML_PATH_ENABLED = 'Webguru_livechat/livechat/enabled';

    const XML_PATH_WIDGET = 'Webguru_livechat/livechat/widget';

    public function __construct(

        \Magento\Framework\App\Config\ScopeConfigInterface $scopeConfig,

        \Magento\Framework\View\Element\Template\Context $context

    )

    {

        $this->scopeConfig = $scopeConfig;

        parent::__construct($context);

    }

    public function isEnabled() {

        $storeScope = \Magento\Store\Model\ScopeInterface::SCOPE_STORE;

        return $this->scopeConfig->getValue(self::XML_PATH_ENABLED, $storeScope);

    }

    public function getWidget()

    {

        $storeScope = \Magento\Store\Model\ScopeInterface::SCOPE_STORE;

        return $this->scopeConfig->getValue(self::XML_PATH_WIDGET, $storeScope);

    }

}

Create:  your_magento_base/app/code/Webguru/LiveChat/view/frontend/layout/default.xml

<?xml version="1.0"?>

<page xmlns:xsi="//www.w3.org/2001/XMLSchema-instance" xsi:noNamespaceSchemaLocation="../../../../../../../lib/internal/Magento/Framework/View/Layout/etc/page_configuration.xsd">

    <body>

        <referenceContainer name="content">

            <block class="Webguru\LiveChat\Block\Adminhtml\LiveChatHelper\LiveChatConfig"

                   name="front_livechat"

                   template="Webguru_LiveChat::livechatwidget.phtml" />

        </referenceContainer>

    </body>

</page>

Create: your_magento_base/app/code/Webguru/LiveChat/view/frontend/templates/livechatwidget.phtml

<?php

if($block->isEnabled()) {

    echo $block->getWidget();

}

?>

Finally, in the terminal, type: sudo rm -rf var

And this is the outcome.

If you choose not to activate it, simply disable it in its Configuration.
