How to Move Magento 2 from Localhost to Server

Published on: February 01, 2023

How to Move Magento 2 from Localhost to Server? In this guide, we will show you the steps involved in How to Transfer Magento 2 from Localhost to server, so you can make your online store accessible to the public. Whether you’re developing your store for the first time or migrating an existing one, moving Magento 2 from localhost to server can be a complex process. However, by following the instructions carefully, you can ensure a smooth and successful transition of your store from a local development environment to a live website.

How to Move Magento 2 from Localhost to Server: A Comprehensive Guide

Step 1: Exporting the Database from Localhost

  • Connect to your local host platform (such as XAMPP or WAMP) and log in to phpMyAdmin.
  • Locate the database for your store.
  • On the top menu, click “Export” and then hit the “Go” button.
  • Download the database file as a .sql extension or save it as a .zip file.

Step 2: Zipping Magento Files

  • Select all the files and folders from localhost.
  • Navigate to the htdocs folder (if using XAMPP) or the www folder (if using WAMP).
  • Go to the Magento folder and compress the files into a zip format.

Step 3: Uploading Magento Files to the Web Server

  • To upload the database, use cPanel on your website server.
  • Log in to cPanel.
  • Access phpMyAdmin.
  • Create a new database by clicking the “New” button.
  • Remember the database name, username, and password for future reference.

Step 4: Importing the Database

After creating the database, import the database file from Step 1.

There are two methods for importing the database:

  • Method 1: Command Line (using the following command: mysql -p -u username database_name < backup_file.sql)
  • Method 2: phpMyAdmin (by accessing the “Import” option on the top menu, selecting the .sql file, and clicking “Choose File”).

Step 5: Uploading Magento Files

You can choose to create a subfolder for a new domain or use the normal directory.

  • Upload the zip file created in Step 2 and extract it.

Step 6: Updating the env.php File for New Database Access:

  • After uploading and extracting the zip folder, access the app/etc/env.php file.
  • In the env.php file, update the information for the new database created on your domain. This includes the username, name, and password used during the import process.
  • Look for the appropriate location in the file to make these changes.

Step 7: Finding the New Domain Name for Magento

  • Save the changes made in Step 6.
  • If you need to edit the file, upload it again to the correct directory.
  • In the Magento database, go to the Table Core_Config_Data.
  • Replace the localhost URL with the domain name in the “web/unsecure/base_url

Step 8: Running Commands

To access your store, run the following commands:

php bin/magento setup:upgrade

php bin/magento setup:static-content:deploy -f

php bin/magento cache:flush

php bin/magento cache:clean

Congratulations! You now know How to Move Magento 2 from Localhost to Server successfully moved your Magento 2 store from localhost to server.

Conclusion

If you’re wondering how to move Magento 2 from localhost to server, you can follow these steps to successfully transfer your online store from a local development environment to a web server. These steps include exporting the database from localhost, uploading the files and the database to the web server, updating the env.php file for new database access, finding the new domain name for your site, and running commands to access your store. By completing these steps, you can ensure a smooth and successful transition of your store from a local environment to a live website. If you are new to it and are looking for a guide on How to Install Magento 2, this process can help you understand the steps involved in setting up a store on a server.
