How to remove Magento 2 extension

Published on: February 03, 2023

In this article, we’ll discuss how to remove Magento 2 extension from your store if you’ve decided to do so. Fortunately, the process of removing a Magento 2 extension is straightforward, and we’ll guide you through the steps on how to uninstall Magento 2 extension successfully. Whether you no longer need a module or want to replace it with a different solution, we’ll show you how to remove the extension from your store. So, let’s dive in!

How to Remove Magento 2 Extension: A Comprehensive Guide

Removal using Composer (simple method)

Hey there, fellow Magento users! Are you wondering how to safely and securely remove an extension from your system? Well, you’re in luck because I have some advice for you!

Firstly, let me just say that the most secure way to install Magento 2.4 is through Composer. So if you haven’t already done that, I highly recommend it. Now, let’s move on to removing an extension.

There are two methods available: simple and complex. The simple method is suitable for extensions with no dependencies or connections to other extensions. However, if the extension you want to remove has dependencies and heavily affects your store, a more complex approach is required. Don’t worry though, I’ll walk you through both methods.

Let’s start with the simple method:

  1. Log into your domain via SSH/CLI and navigate to your store’s root.
  2. Run the command “bin/Magento module: status” to get a list of enabled modules and find the internal name of the module you want to remove (e.g. Webguru_Demo).
  3. Run the command “bin/Magento module: disable <Internal Module Name>” to disable the module.
  4. Run the command “bin/magento setup: upgrade” to remove the module from all store activities.
  5. Go to the root directory of the module you want to remove and find the “composer.json” file. In our example, it’s located at “/vendor/Webguru/module-demo/composer.json.” Find the “composer name” in the “name” attribute.
  6. Return to the store’s root and run the command “composer remove <composer name>” (e.g. composer remove Webguru/module-demo).

Removing with Composer (complex)

How to remove Magento 2 extension depends on the complexity of the module. Complex removal is necessary for modules with dependencies or when the module is deeply integrated into your store, requiring maintenance mode during removal. it simplifies this process with a specific command:

1.Log into your domain via SSH/CLI and navigate to your store’s root.

2. Run the command “bin/Magento module: status” to get a list of enabled modules and find the internal name of the module you want to remove (e.g. Webguru_Demo).

3. Run the command “bin/Magento module: uninstall [options] <internal module name>” .

4. To automatically backup the database, add the option “–backup-db”. The tar-gzipped backup file will be located in the “/var/backup” directory.

5. To automatically backup all store media (in the “/pub/media” directory), add the option “–backup-media”. This will also be placed in the “/var/backup” directory.

6. To automatically backup the store code (useful for restoring the system after incorrect removal), add the option “–backup-code”. This will backup the entire store excluding temporary or generated directories and will also be placed in the “/var/backup” directory.

7. To automatically remove all tables created by the module, add the option “–remove-data”.

8. To automatically clean static files after removal, add the option “–clear-static-content”.

9. For example, if we want to backup the database and regenerate static content, the full command in our example would be “bin/magento module:uninstall –backup-db –clear-static-content Magento_Demo”

The command will halt if it encounters any dependencies and displays them to you.

bin/magento module:uninstall Webguru_Demo

Cannot uninstall module ‘Magento_Demo’ because the following module(s) depend on it:

Webguru_SampleData

In such cases, simply add the dependencies before the module name. The command would then look like this: bin/Magento module: uninstall –backup-db –clear-static-content Magento_SampleData Magento_Demo.

This command will automatically switch your store to maintenance mode, deactivate and remove the application, and clear the cache and data specified in the options. It will then run the composer removal command and upon completion, turn off the maintenance mode.

Manual Removal

If your store does not use Composer, you can remove the extension manually with the following steps:

1.Log in via SSH/CLI to your domain and navigate to the root of your store.

2. Run the command bin/Magento module: status to see a list of enabled modules, and identify the internal name of the module you want to remove (e.g. Webguru_Demo).

3. Run the command bin/Magento module: disable <Internal Module Name> (e.g. bin/Magento module: disable Webguru_Demo) to disable the module.

4. Run the command bin/magento setup: upgrade to remove the module from all store activities.

5. Remove the module directory with the command rm -rf <Module Directory>.

6. Clean the cache with bin/magento cache: clean and regenerate the static content with bin/magento setup:static-content: deploy -f.

Note: This manual removal method may result in unchecked errors, so it’s recommended to use Composer for uninstall extension Magento 2. By using Composer, you can ensure that all the dependencies of the extension are removed properly and there are no conflicts with other extensions.

Conclusion

In conclusion, the best and safest way to remove extensions is by using Composer. For simple extensions, the process is straightforward and can be accomplished by following a few steps. However, for complex extensions with dependencies, a more complex approach is recommended, This command automatically puts the store in maintenance mode, disables and removes the application, cleans the cache and all specified data, and runs the composer removal command. If you are wondering how to remove Magento 2 extension manually and your store does not use Composer, you can try to remove the extension manually, but this may result in unchecked errors. It is always recommended to use Composer for extension removal in it.
