How to setup Google Tag Manager in Magento 2 store

Published on: February 08, 2023

How to setup Google Tag Manager in Magento 2 store – Tracking visitor behavior on your online store is crucial for boosting profits. By identifying pages where customers abandon their carts, you can eliminate the issue and increase sales. Google Tag Manager (GTM) is a popular service used to gather customer journey data through small event tracking. GTM is managed through a web interface and must be configured to track visitor steps. This configuration requires setting up your store, GTM service, and analytics service (such as Google Analytics 4). A guide to setting up GTM in Magento 2 store is available for reference.

How to Setup Google Tag Manager in Magento 2 Store: A Step-by-Step Guide

What is Google Tag Manager

Google Tag Manager (GTM) is a powerful tool that can be used to track website activity, including eCommerce transactions, and can be integrated with Magento 2 using a Magento 2 Google Tag Manager module. Tracking website activity typically requires a specific code for each event to be tracked, which can make tracking a complex and time-consuming process involving significant website changes. However, GTM simplifies this process by allowing you to manage tracking tags through a user-friendly web interface, without the need for extensive website modifications.

To set up Google Tag Manager in your Magento 2 store, you can use the Magento 2 Google Tag Manager module. How to setup Google Tag Manager in Magento 2 store is a straightforward process that only requires adding two GTM code snippets. With this integration, businesses can gain valuable insights into user behavior, allowing them to optimize their marketing strategies, increase conversion rates, and ultimately, boost revenue.

By using the Magento 2 Google Tag Manager module, you can seamlessly integrate GTM with your store without having to modify your store’s theme templates. This makes the integration process even simpler and more efficient. If you’re looking to take your eCommerce efforts to the next level, integrating Magento 2 with GTM is a crucial step that can provide significant benefits for your business.

Why a GTM extension for Magento is recommended

To learn how to setup Google Tag Manager in Magento 2 store, you must add GTM code snippets to your store’s frontend theme. Adobe Commerce edition of Magento offers a built-in interface for adding GTM code. However, Magento Open Source stores need to use a third-party extension, such as the “Google Tag Manager & Facebook Pixel for Magento 2,” which supports the Hyva theme and comes with preconfigured GTM tags, as well as support for the Facebook Pixel tracker. A guide to installing the extension is available, which can help you integrate GTM with your Magento 2 store and start gaining valuable insights into user behavior.

Note: The built-in Google Tag Manager module will be available in Magento 2.4.5, starting from August 9, 2022.

Instructions for Setting Up GTM in Magento 2

This guide covers the GTM setup for the Magento Open Source edition. The “Google Tag Manager & Facebook Pixel for Magento 2” extension is used to connect to the Google Tag Manager service. To use the GTM Magento extension, you need to provide three identifiers: GTM Account ID, GTM Container ID, and either Google Analytics 4 Measurement ID or Google Analytics 3 Tracking ID.

Step 1: Establish a GTM Account and Container.

  • Go to the Google Tag Manager website and log in with your Gmail credentials. 
  • Click the “Create Account” button, 
  • provide a name for the account in the “Account Setup” section, and select your country.
  • Set up the account container by specifying a name and selecting the target platform. Choose “Web” if you are collecting data from a website. 

Note: To delete a container, go to Admin -> Container Settings -> More (three vertical dots) -> Settings -> Delete. Google provides a 30-day window for restoration. After that, the container will be permanently deleted.

Step 2: Retrieve your GTM Account ID and Container ID.

To retrieve the GTM Container ID, go to the GTM homepage, click on the container name, and find the Container ID in the browser address bar after “containerId=”.

Make note of the Container ID, as you will need to enter it into your Magento store. 

Note: The required Container ID is a 10-digit number, not the formatted “GTM-N48BDMV” version shown in the GTM interface.

To retrieve the GTM Account ID, 

open your container and look at the URL of the current page in the browser address bar. The GTM Account ID can be found after “accounts” in the URL and should be noted as you will need to enter it into your Magento store.

Step 3: Retrieve Google Analytics 4 Measurement ID

To obtain the Google Analytics 4 Measurement ID, log in to your Google Analytics account and navigate to the tracking code setup. The Measurement ID can be found in the code provided and should be written down for later use in the GTM setup process.

  • Access GA4 property for Magento tracking
  • In the Properties & Apps section, select the Google Analytics 4 property used for tracking events in your Magento store.

To find your Google Analytics 4 Measurement ID:

If your GA4 property has received no data, go to the Properties&Apps block and select the GA4 property you use to track events in your Magento store. The Measurement ID will be immediately visible.

If your GA4 property is already receiving data, go to Admin > Property column > Data Streams > Web data stream. Write down the Measurement ID, as you will need to paste it into your Magento store.

Step 4: Obtain the GTM Installation Code Snippet

Go to the GTM home page and select the container you want to use.

On the container page, click on the container ID to open the “Install Google Tag Manager” window.

Copy the code snippets for the <head> and <body> tags provided in the window.

Step 5. Add GTM Code and IDs to Magento Store

Go to Stores > Configuration > Mirasvit Extensions > Google Tag Manager and set Enable GTM to Yes. Then, paste the <head> code snippet in the GTM Regular Code field and the <body> code snippet in the GTM No-Script Code field.

In the Container Import section, paste the previously obtained GTM Account ID, GTM Container ID, and Analytics Measurement ID into their respective fields.

Step 6. Create GTM Tags

To track events, you need to create separate tags and triggers within the GTM service. Start by importing the container information from the GTM Magento extension into your GTM account. In Magento, click the Download Import File for Google Analytics 4 button and save the JSON file.

In GTM, go to the homepage, and click the gear icon. Select the Import container, choose the JSON import file, workspace (usually the Default Workspace), and the import method (overwrite or merge). Preview and confirm the import to see the number of new GTM tags, tag triggers, and tag variables added to the container.

Creating a New GTM Tag:

In the GTM container, navigate to the Workspace and click the New Tag card. Provide a descriptive name for the tag, such as “Tag Type – App Name – Detail.” Then, click Tag Configuration.

Select the GTM tag type, such as GA4 Event. Set the Configuration Tag, for example, GA4 – Configuration, and choose the Event Name from the list. Save the tag and add a trigger when prompted.

If you imported a set of triggers from the GTM Magento extension, you will have a list of available triggers. Otherwise, you need to create a new trigger by clicking the + sign.

Creating a New GTM Tag Trigger:

Click the + sign, then the Trigger Configuration. Choose the Trigger Type, such as 

Page View : 

  • Consent Initialization,
  • Initialization,
  • DOM Ready, 
  • Window Loaded 

Click :

  • Element Visibility,
  • Form Submission, 
  • Scroll Depth, 
  • YouTube Video, 

Other :

  • Custom Event.
  • History Change
  • JavaScript Error
  • Timer
  • Trigger Group

Next, choose where the trigger will activate. Each trigger type has different activation options. For instance, a Page View trigger can be set to activate on all page views or only on certain page views.

Step 7. Activate GTM tags

Activate the tags by clicking the Submit button and selecting Submission Configuration as Publish and Create Version. Enter a Version Name and, if desired, a Version Description, then click Publish.

Click on the three vertical dots (More) and choose to Publish version to receive a message that the published version is live.

Step 8. Check GTM tag functionality

After creating the tags, it is important to verify that they are working correctly to avoid any configuration mistakes.

Locate the latest live version of the tag, click on the three vertical dots (More), and select Preview.

You will be redirected to the Tag Assistant web page. Enter the URL of your store and click Connect.

Your store will open in a new browser window, and if the GTM service is successfully connected to your store, you will see the Tag Assistant Connected window.

Click Continue in the Tag Assistant.

You will then see a page that monitors the performance of your tags. On your store, perform actions that should fire the tag you want to test.

For example, to test the view_item tag, navigate to a product page on your store. The Tag Assistant should show that the tag has been executed.

Verify that your Google Analytics 4 is receiving the tracking data by navigating to the Realtime report and observing the activity on the site in the last five minutes.”
