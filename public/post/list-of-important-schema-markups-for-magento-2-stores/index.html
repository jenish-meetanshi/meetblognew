List of Important Schema Markups for Magento 2 Stores

Published on: October 04, 2024

Schema markup, a structured data vocabulary, plays a pivotal role in enhancing your Magento 2 store’s search engine visibility and user experience.

By providing search engines with clear information about your products, services, and website content, you can improve your chances of appearing in rich snippets and knowledge graphs.

Understanding Schema Markup

Schema markup is a standardized format that allows you to provide search engines with detailed information about your website’s content. It helps search engines understand the context of your pages and display relevant information in search results. For example, product schema markup can help your product listings appear with product images, prices, and reviews in search results.

Why is Schema Markup Important for Magento 2 Stores?

  1. Improved Search Engine Rankings: Search engines prioritize websites that use schema markup as it helps them understand the content better. This can lead to higher search engine rankings and increased organic traffic.
  2. Enhanced Click-Through Rates: Rich snippets and knowledge graphs make your listings more visually appealing and informative, increasing users’ likelihood of clicking on them.
  3. Better User Experience: Schema markup provides users with relevant information upfront, reducing the need to navigate to your website to find what they’re looking for.

Essential Schema Markups for Magento 2 Stores:

1. Product Schema:

  • Key properties: name, description, image, price, availability, brand, SKU, offers, and more.
  • Benefits: Helps your products appear with rich snippets like images, price, availability, and ratings in search results, enhancing visibility and click-through rates.
  • Example:
{
  "@context": "http://schema.org",
  "@type": "Product",
  "name": "Blue Denim Jeans",
  "description": "Classic blue denim jeans with a regular fit.",
  "image": "https://example.com/images/jeans.jpg",
  "price": "49.99",
  "availability": "InStock"
}

2. Article Schema:

  • Key properties: headline, author, datePublished, dateModified, articleBody, publisher, image, keywords, and more.
  • Benefits: Helps your blog posts and articles appear with a featured image, author, and publication date in search results.
  • Example:
{
"@context": "http://schema.org",
"@type": "Article",
"headline": "Magento 2 SEO Tips",
"author": "John Doe",
"datePublished": "2023-01-01",
"image": "https://example.com/images/blog-post.jpg"
}

3. Organization Schema:

  • Key properties: name, logo, address, contactPoint, sameAs, url, description, foundingDate, founders.
  • Benefits: Enhances the visibility of your organization by displaying structured information like business name, contact details, and social profiles in search results.
  • Example:
{
"@context": "http://schema.org",
"@type": "Organization",
"name": "Acme Corporation",
"logo": "https://example.com/images/logo.png",
"address": {
"@type": "PostalAddress",
"street address": "123 Main Street",
"city": "Anytown",
"postalCode": "12345"
}
}

4. LocalBusiness Schema:

  • Key properties: name, address, priceRange, telephone, openingHours, url, geo, rating, menu.
  • Benefits: Helps local businesses rank better in local searches and allows search engines to show relevant business information like hours, pricing, and location.
  • Example:
{
"@context": "http://schema.org",
"@type": "LocalBusiness",
"name": "Acme Restaurant",
"address": {
"@type": "PostalAddress",
"streetAddress": "123 Main Street",
"city": "Anytown",
"postalCode": "12345"
},
"priceRange": "$"
}

5. Breadcrumb Schema:

  • Key properties: itemListElement (contains a list of pages in the breadcrumb trail with their positions, URLs, and names).
  • Benefits: Helps search engines understand your website’s structure and improves navigation for users, allowing breadcrumbs to be shown in search results.
  • Example:
{
"@context": "http://schema.org",
"@type": "BreadcrumbList",
"itemListElement": [
{
"@type": "ListItem",
"position": 1,
"item": {
"@type": "WebPage",
"url": "https://example.com/",
"name": "Home"
}
},
{
"@type": "ListItem",
"position": 2,
"item": {
"@type": "WebPage",
"url": "https://example.com/products/",
"name": "Products"
}
}
]
}

6. FAQ Schema:

  • Key properties: mainEntity (contains a list of questions and their accepted answers).
  • Benefits: Helps FAQs appear in an enhanced format on search results, showing the question and its corresponding answer directly in the results, improving SEO and user engagement.
  • Example:
{
  "@context": "http://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "What is the return policy for Blue Denim Jeans?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "You can return the Blue Denim Jeans within 30 days of purchase for a full refund."
      }
    },
    {
      "@type": "Question",
      "name": "Do the Blue Denim Jeans come in other sizes?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Yes, the Blue Denim Jeans are available in sizes ranging from 28 to 40."
      }
    }
  ]
}

How to Implement Schema Markup in Magento 2

Implementing schema markup in Magento 2 typically involves either modifying theme files or using third-party extensions designed for structured data management.

  1. Using Third-Party Extensions: Many extensions available on the Magento Marketplace simplify the process of adding structured data without requiring extensive coding knowledge. Through this schema markup extension, you can add many types of schema. These tools often provide user-friendly interfaces for configuring various schemas.
  2. Manual Implementation: If you prefer manual implementation, you will need some coding skills:
    • Navigate to your theme directory and locate the relevant template files.
    • Add JSON-LD structured data scripts within these files using PHP variables to generate product information dynamically.
    • Test your implementation using Google’s Rich Results Test tool to ensure everything functions correctly.

Testing Your Schema Markup

After implementing schema markup, it’s crucial to test it thoroughly:

  • Use the Google Rich Results Test tool to check if your structured data is valid.
  • Monitor the performance of your rich snippets through Google Search Console under the “Enhancements” section.
